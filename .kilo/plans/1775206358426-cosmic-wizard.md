# SmileX Dict 产品功能需求调研报告

## 产品定位

SmileX Dict 是一个面向英语学习者（尤其英语专业学生）的多端背单词与文章练习应用，支持 Web/PWA、移动端（Capacitor）和桌面端（Electron）。技术栈为 React 19 + Redux Toolkit + Tailwind CSS + FastAPI + SQLite。

## 三、需要新增的功能（按优先级排序）

### P0 - 核心缺失功能（产品可用性必备）

#### 3.1 真实词典数据
- **现状**: 推荐词典（CET-4/6/TOEFL/IELTS）仅有名称和计数，实际词列表为空
- **需求**: 导入真实词库数据，至少覆盖 CET-4（约4000词）
- **方案**: 
  - 后端添加词库数据导入接口（支持批量导入）
  - 提供 CSV/JSON 格式词库模板
  - 预置常用词库数据（CET-4、CET-6、TOEFL、IELTS）

#### 3.2 单词增删改查完善
- **现状**: 只有 list 和 create，无 update 和 delete
- **需求**:
  - `PUT /api/words/:id` - 修改单词释义、例句等
  - `DELETE /api/words/:id` - 删除单词
  - `DELETE /api/dicts/:id` - 删除词典
  - `PUT /api/dicts/:id` - 修改词典信息
  - 前端批量操作（批量标记掌握/删除）

#### 3.3 数据导入导出
- **现状**: 无任何导入导出功能
- **需求**:
  - 导出学习数据（JSON/CSV）用于备份
  - 导入词典数据（CSV 格式：term, meaning, ipa, example）
  - 导出/导入用户全部学习进度
  - 与 Anki 格式兼容的导入

### P1 - 重要功能（提升用户体验）

#### 3.4 搜索功能
- **现状**: 无搜索功能
- **需求**:
  - 词典内搜索单词
  - 全局搜索（跨词典搜索单词）
  - 文章搜索
  - 搜索历史

#### 3.5 发音功能
- **现状**: PronunciationExercise 组件存在但使用模拟数据
- **需求**:
  - 集成 Web Speech API 或第三方 TTS 服务进行单词发音
  - 音标 IPA 播放
  - 例句朗读
  - （远期）语音识别评估用户发音

#### 3.6 面板增强
- **现状**: 面板功能基础，historyStats 从未填充
- **需求**:
  - 历史统计接口 `GET /api/stats/history?days=N`
  - 时间范围筛选（7天/30天/180天）
  - 堆叠柱状图（新词/复习/错词/默写分类统计）
  - 复习预测（未来7天预计复习量）
  - 错词统计卡片

#### 3.7 离线模式
- **现状**: 必须联网才能使用
- **需求**:
  - Service Worker 缓存静态资源
  - IndexedDB 存储离线学习数据
  - 离线时学习，联网后同步
  - 离线状态提示

### P2 - 增强功能（提升竞争力）

#### 3.8 暗黑模式
- **现状**: 无深色主题
- **需求**:
  - Tailwind `darkMode: 'class'` 配置
  - 跟随系统偏好 + 手动切换
  - 在设置页提供开关

#### 3.9 键盘快捷键
- **现状**: 练习页无键盘快捷键
- **需求**:
  - 练习页：Enter 提交、Space 下一题、数字键选择选项
  - 全局：Ctrl+K 搜索
  - 显示快捷键提示

#### 3.10 词汇网络可视化
- **现状**: VocabularyNetwork 组件使用简单 SVG，无真实数据
- **需求**:
  - 基于同义词/反义词构建词义关系图
  - 可交互的图谱展示（缩放、拖拽）
  - 点击节点跳转到单词详情

#### 3.11 学习路径与个性化推荐
- **现状**: LearningPath 组件存在但 milestone 为空
- **需求**:
  - 基于学习目标和时间创建学习计划
  - 里程碑追踪
  - 薄弱环节识别与推荐
  - 智能推荐今日应学内容

#### 3.12 错词模式分析
- **现状**: 无错误模式分析
- **需求**:
  - 识别拼写错误模式（如 -tion/-sion 混淆）
  - 语义混淆分析（近义词选错）
  - 薄弱领域识别
  - 针对性练习推荐

### P3 - 优化功能（锦上添花）

#### 3.13 PWA 完善
- **现状**: vite-plugin-pwa 已安装但未配置
- **需求**:
  - 配置 Service Worker
  - 生成 PWA icons（192x192, 512x512）
  - 完善 manifest.json
  - 支持安装到桌面

#### 3.14 Electron 增强
- **现状**: electron/main.js 仅有基础代码
- **需求**:
  - 添加 preload.js（contextBridge 安全通信）
  - 窗口状态持久化（位置、大小）
  - 系统托盘（最小化到托盘）
  - 自动更新（electron-updater）
  - 应用菜单

#### 3.15 社交/分享功能
- **需求**:
  - 学习打卡分享（生成图片）
  - 词典分享给其他用户
  - 学习排行（可选）
  - 学习小组/协作

#### 3.16 通知提醒
- **需求**:
  - 复习提醒推送
  - 每日学习目标提醒
  - 浏览器 Notification API
  - 移动端推送通知

## 八、总结

SmileX Dict 已具备**扎实的学习核心**（SM-2 算法、多模式练习、文章练习、用户认证），但在以下方面存在明显短板：

1. **内容缺失**: 推荐词典无实际词库数据，是最紧迫的问题
2. **交互不完整**: 缺少发音、搜索、快捷键等基础交互
3. **数据管理弱**: 无法导入/导出/删除数据
4. **稳定性问题**: 多处 Bug 影响使用体验（responseTime 计算、SM-2 循环等）
5. **安全风险**: 硬编码密钥、CORS 通配符等

建议按 Phase 1 → Phase 2 → Phase 3 的顺序推进，优先解决内容缺失和基础 Bug，再逐步完善体验和高级功能。
