# SmileX Dict 功能改进与界面优化计划

## 一、头脑风暴：功能缺失与改进分析

通过对现有代码库的全面审查，识别出以下关键问题和改进方向：

### A. 功能性缺陷（必须修复）

1. **Dicts 页面统计数据硬编码为 0** (`src/routes/Dicts.tsx:237-239`)
   - "新词/复习/默写" 三个统计卡片始终显示 0，未对接真实数据
   - 需要根据当前词典从 `words` 状态计算真实值

2. **PracticeArticles 未携带认证 Token** (`src/routes/PracticeArticles.tsx:41,86`)
   - 使用裸 `fetch` 调用 `/api/stats/event`，未注入 Bearer Token
   - 导致统计事件在服务端认证失败

3. **模态框关闭按钮未国际化** (`src/routes/Dicts.tsx:331,441,559,579`)
   - 多个模态框的"关闭"按钮硬编码中文，未使用 `t()` 翻译

4. **单词进度不同步服务端** 
   - SM-2 学习数据（interval, efactor, repetition 等）仅存在 Redux 本地
   - 后端有 `user_word_progress` 表但前端未与后端同步
   - 切换设备或清除浏览器数据后进度丢失

5. **无单词编辑/删除 UI**
   - 后端 API 支持 `PUT /api/words/{id}` 和 `DELETE /api/words/{id}`
   - 前端收藏/错词/已掌握页面没有编辑和删除入口

6. **无词典删除 UI**
   - 后端支持 `DELETE /api/dicts/{id}`，前端无对应操作

7. **无文章删除/编辑功能**
   - 文章库页面只有列表展示，没有删除或编辑按钮
   - 后端支持 `DELETE /api/articles/{id}`

### B. 功能增强（提升用户体验）

8. **学习进度条**
   - 练习页面没有进度指示器（如 "第 12/50 个单词"）
   - 应增加环形或线性进度条

9. **个性化首页**
   - 已登录用户的首页仍显示通用介绍
   - 应显示：今日任务完成度、待复习数量、学习连续天数等

10. **单词详情页完善**
    - 收藏/错词/已掌握列表中的单词卡片缺少发音按钮
    - 缺少词性标签展示
    - 缺少例句翻译

11. **自动备份提醒**
    - 设置页面有手动导出，但没有自动备份提醒
    - 建议每周提醒一次数据备份

12. **快捷键帮助面板**
    - Ctrl+K 搜索已有，但缺少全局快捷键帮助面板（按 ? 显示）

13. **学习成就系统**
    - 连续签到天数、总掌握词数、总练习时间等成就徽章
    - 提升用户粘性

### C. UI/界面优化

14. **页面过渡动画统一**
    - 当前使用 `page-enter` 类，各页面一致
    - 但缺少页面间切换的方向感（前进/后退动画）

15. **空状态优化**
    - 部分空状态显示 emoji（如 📚📖📝），应确保跨平台一致
    - 建议用 SVG 图标替代 emoji，保持视觉一致性

16. **响应式优化**
    - Dicts 页面词典网格 `md:grid-cols-4` 在中等屏幕下可能过挤
    - 文章练习的输入区域在移动端需要更大的触控区域

17. **Toast 动画**
    - 当前 Toast 组件缺少入场/出场动画

18. **移动端导航优化**
    - 已登录用户在移动端看不到"面板"、"收藏"等快捷入口
    - 建议在移动端增加常用功能快捷入口

19. **深色模式细节**
    - 部分 `glass-card` 在深色模式下对比度不足
    - `bg-gray-50/80 dark:bg-gray-700/30` 等在某些元素上过暗

## 三、具体实施步骤

### Step 1: 修复 Dicts 页面统计数据
**文件**: `src/routes/Dicts.tsx`

将硬编码的 0 替换为从 Redux store 的 words 数据中计算的真实统计：
- 新词数：根据当前词典的 `status === 'new'` 单词数
- 待复习数：根据 `nextReviewDate <= today` 且 `status !== 'mastered'` 的数量
- 默写数：从今日统计中获取

### Step 2: 修复 PracticeArticles Token 问题
**文件**: `src/routes/PracticeArticles.tsx`

将 `fetch(API_BASE + '/api/stats/event', ...)` 替换为使用 `statsApi.addEvent()`，自动携带 Token。

### Step 3: 国际化修复
**文件**: `src/routes/Dicts.tsx`, i18n locales

- 将所有硬编码 "关闭" 替换为 `t('common.close')`
- 在 i18n zh.json/en.json 中添加 `common.close` 翻译
- About 页面 "Tech Stack" 标题使用 `t('about.techStack')`

### Step 4: Toast 动画
**文件**: `src/components/Toast.tsx`, `src/index.css`

- Toast 入场：从顶部滑入 + 淡入
- Toast 出场：向上滑出 + 淡出
- 在 `index.css` 添加 `@keyframes toastIn` / `toastOut`

### Step 5: WordList 增加编辑/删除
**文件**: `src/components/WordList.tsx`

- 每个单词卡片增加"编辑"和"删除"图标按钮
- 编辑功能：弹出模态框编辑单词拼写、释义、例句
- 删除功能：确认后调用 `wordsApi.delete()` 并从 Redux 中移除

### Step 6: Dicts 页面词典删除
**文件**: `src/routes/Dicts.tsx`

- 自建词典卡片增加删除按钮（收藏/错词/已掌握不可删除）
- 删除前弹出确认模态框
- 调用 `dictsApi.delete()` 并刷新列表

### Step 7: Library 页面文章操作
**文件**: `src/routes/Library.tsx`

- 每个文章卡片增加"练习"和"删除"按钮
- 删除需确认模态框
- 调用 `articlesApi.delete()` 并从 Redux 中移除

### Step 8: 个性化首页
**文件**: `src/routes/Home.tsx`

已登录用户看到的首页内容替换为：
- 今日任务完成卡片（新学 N/M 词、复习进度、连续签到天数）
- 快速操作按钮（开始练习、复习错词、查看面板）
- 最近学习的单词预览

### Step 9: 练习页进度条
**文件**: `src/routes/PracticeWords.tsx`

在页面顶部增加一个细进度条：
- 显示 `(index + 1) / queue.length` 的百分比
- 使用 `bg-brand-500` 带渐变动画

### Step 10: 深色模式对比度修复
**文件**: 多处

审查并修复：
- `dark:bg-gray-700/30` → `dark:bg-gray-700/50` 提升可见度
- 卡片内文字颜色确保 WCAG AA 标准
- 图标颜色在深色模式下的可见性

## 四、验收标准

1. Dicts 页面统计数据正确反映当前学习状态
2. 文章练习的统计事件能成功上报服务端
3. 所有用户可见文本已国际化
4. 单词/词典/文章支持完整 CRUD 操作
5. 已登录用户首页显示个性化内容
6. 练习页显示进度条
7. 深色模式下所有元素清晰可辨
8. 移动端操作流畅，无溢出或遮挡
