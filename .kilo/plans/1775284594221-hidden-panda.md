# 权限和导入词典问题修复计划

## 问题概述

经过代码分析,发现以下主要问题:

### 1. 前后端数据不同步 - 词典创建
- **前端** (Dicts.tsx:42-46): 使用本地Redux状态创建词典,仅在Redux中添加,不调用后端API
- **后端** (main.py:241-261): 需要调用 `dictsApi.create` 才能在数据库中创建词典
- **结果**: 前端创建的词典在后端不存在,导致后续导入词表时404错误

### 2. 导入词表权限问题
- **Dicts.tsx (100-104)**: 选择本地Redux创建的词典ID进行导入
- **后端 (main.py:1210-1219,1317-1326)**: 验证词典是否在数据库中存在且属于当前用户
- **结果**: 前端词典ID在后端不存在,返回404错误

### 3. 未登录用户导入问题
- **前端**: 虽然显示访客模式警告,但仍然允许导入操作
- **后端**: 所有导入API都需要认证 (`get_current_user` 依赖)
- **结果**: 未登录用户导入会失败,前端没有友好提示

### 4. 后端词典ID生成问题
- **后端 (main.py:247-256)**: 使用 `id=f"d{count + 1}"` 生成ID
- **问题**:
  - 不同用户可能生成相同ID (都从 `d1` 开始)
  - 与前端特殊词典ID (`collected`, `wrong`, `mastered`) 可能冲突

### 5. Settings页面导入无词典关联
- **Settings.tsx (165-168)**: 调用 `lookupApi.lookup(term, true)` 不指定 `dictId`
- **后端 (main.py:1061)**: `dictId=dictId` 参数为 `None`
- **结果**: 导入的单词没有关联到任何词典

### 6. 缺少数据同步机制
- **登录后**: 没有从后端加载用户的词典列表
- **词典操作**: 没有同步到后端

### 第三阶段: dictsSlice 增强

**文件**: `src/features/dicts/dictsSlice.ts`

**修改点**:
1. 添加新的actions来处理后端词典列表
   - `setDicts`: 设置完整词典列表
   - `addServerDict`: 添加单个后端词典

**实现**:
```typescript
reducers: {
  // ... existing reducers
  setDicts(state, action: PayloadAction<DictItem[]>) {
    // 保留特殊词典,添加后端词典
    const specialIds = ['collected', 'wrong', 'mastered']
    const specialDicts = state.mine.filter(d => specialIds.includes(d.id))
    state.mine = [...specialDicts, ...action.payload.filter(d => !specialIds.includes(d.id))]
  },
  addServerDict(state, action: PayloadAction<DictItem>) {
    const exists = state.mine.find(d => d.id === action.payload.id)
    if (!exists) {
      state.mine.push(action.payload)
    }
  },
}
```
