# 优化后端日志全局配置 — 增加开发/生产环境区分

## 现状分析

当前日志配置极为简单，仅在 `server/app/main.py` 中调用 `logging.basicConfig()`，存在以下不足：

- 无环境区分（开发/生产使用相同配置）
- 无文件日志输出（仅 stdout）
- 无日志轮转策略
- 无结构化日志（JSON 格式）支持
- 无按模块/命名空间细化日志级别
- `config.py` 中缺少 `APP_ENV` 环境变量

## 实施计划

### 1. 在 `server/app/config.py` 中增加环境相关配置字段

- 新增 `APP_ENV: str = "development"` 字段（可选值：`development` / `production`）
- 新增辅助属性 `is_debug` / `is_production`，便于代码中快速判断环境
- 更新 `.env.example` 添加 `APP_ENV=development`

### 2. 创建独立日志配置模块 `server/app/logging_config.py`

基于 `APP_ENV` 环境变量，使用 Python 标准 `logging.config.dictConfig` 提供两套配置：

#### 开发环境 (`development`)
- **格式**: 彩色、人类可读格式：`时间 级别 [模块名] 消息`
- **级别**: `DEBUG`（默认）
- **输出**: 仅 `stdout` 控制台
- **第三方库日志级别**: `WARNING`（抑制 SQLAlchemy、uvicorn 等噪音）

#### 生产环境 (`production`)
- **格式**: 结构化 JSON（包含 `timestamp`、`level`、`logger`、`message`、`environment` 字段）
- **级别**: `INFO`（默认，可通过 `LOG_LEVEL` 覆盖）
- **输出**: `stdout` + 文件（`logs/app.log`，10MB 轮转，保留 5 个备份）
- **第三方库日志级别**: `WARNING`
- **错误日志单独文件**: `logs/error.log`（仅记录 `ERROR` 及以上级别）

两套配置均支持通过 `LOG_LEVEL` 环境变量覆盖默认级别。

### 3. 更新 `server/app/main.py`

- 移除内联 `logging.basicConfig()` 调用
- 导入并调用新的 `setup_logging()` 函数（从 `logging_config.py`）
- 保持 `logger = logging.getLogger("app")` 不变

### 4. 更新 `.env.example`

添加以下新字段：
```env
# 环境配置
APP_ENV=development
```

### 5. 创建 `server/logs/.gitkeep`

确保 `logs/` 目录被 Git 跟踪（生产环境日志文件的目标目录），并在 `server/.gitignore` 中添加 `logs/*.log` 忽略规则。

## 文件变更清单

| 文件 | 操作 |
|------|------|
| `server/app/config.py` | 修改 — 增加 `APP_ENV` 字段 |
| `server/app/logging_config.py` | 新建 — 日志配置模块 |
| `server/app/main.py` | 修改 — 替换 basicConfig 为 setup_logging() |
| `server/.env.example` | 修改 — 增加 `APP_ENV` |
| `server/logs/.gitkeep` | 新建 — 确保 logs 目录存在 |
| `server/.gitignore` | 修改/新建 — 忽略 `logs/*.log` |

## 不变的部分

- 各模块中 `logger = logging.getLogger(...)` 的调用无需改动
- 不引入第三方日志库（`loguru`、`structlog` 等），使用 Python 标准 `logging` 模块 + `json` 序列化即可
- 现有环境变量 `LOG_LEVEL` 继续生效，作为级别覆盖
