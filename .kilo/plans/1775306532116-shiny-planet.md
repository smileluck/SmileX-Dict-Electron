# 登录后记录 Cookies — Capacitor/Electron 支持

## 概述

当前认证 token 存储在 `localStorage` 中。在 Capacitor（iOS/Android）和 Electron 环境中，`localStorage` 可能在某些场景下不可靠（如 WebView 清理、应用更新等）。需要将 token 同时存储到 cookie 中以增强持久性。

## 现状分析

- **前端 token 管理**：`src/services/api.ts` 通过 `localStorage` 存取 token（key: `smilex_dict_token`）
- **Redux 状态**：`src/features/auth/authSlice.ts` 初始化时从 `localStorage` 读取 token
- **后端认证**：纯 JWT Bearer token，不使用 cookie
- **Electron**：`contextIsolation: true`, `sandbox: true`，preload 暴露了 `isElectron` 标志
- **Capacitor**：基础配置，无额外插件

## 方案设计

采用 **前端 cookie 辅助存储** 方案：

1. 登录/注册成功后，**同时**将 token 写入 `document.cookie` 和 `localStorage`
2. 初始化读取时，优先从 `localStorage` 读取，若不存在则从 cookie 兜底读取
3. 登出时，同时清除 `localStorage` 和 cookie
4. cookie 设置合理的 `path=/` 和 `max-age`（7天，与 token 过期一致）

> 不采用 HttpOnly cookie 方案，因为当前后端是纯 REST API + JWT，改动太大。前端 cookie 方案只需修改前端代码，兼容现有架构。

## 实现步骤

### 1. `src/services/api.ts` — 添加 cookie 读写函数

添加三个工具函数：

```typescript
function setCookieToken(token: string): void {
  document.cookie = `${TOKEN_KEY}=${token}; path=/; max-age=${TOKEN_EXPIRY_MS / 1000}; SameSite=Lax`
}

function getCookieToken(): string | null {
  const match = document.cookie.match(new RegExp('(?:^|; )' + TOKEN_KEY.replace(/([.$?*|{}()\[\]\\\/+^])/g, '\\$1') + '=([^;]*)'))
  return match ? decodeURIComponent(match[1]) : null
}

function removeCookieToken(): void {
  document.cookie = `${TOKEN_KEY}=; path=/; max-age=0`
}
```

### 2. `src/services/api.ts` — 修改 `setToken` 函数

同时写入 `localStorage` 和 cookie：

```typescript
function setToken(token: string): void {
  if (token) {
    localStorage.setItem(TOKEN_KEY, token)
    setCookieToken(token)
  } else {
    localStorage.removeItem(TOKEN_KEY)
    localStorage.removeItem(TOKEN_EXPIRY_KEY)
    removeCookieToken()
  }
}
```

### 3. `src/services/api.ts` — 修改 `getToken` 函数

增加 cookie 兜底读取：

```typescript
function getToken(): string | null {
  const token = localStorage.getItem(TOKEN_KEY)
  if (token) return token
  // Fallback: read from cookie (important for Capacitor/Electron)
  const cookieToken = getCookieToken()
  if (cookieToken) {
    localStorage.setItem(TOKEN_KEY, cookieToken)
  }
  return cookieToken
}
```

### 4. `src/services/api.ts` — 修改 `clearAuthStorage` 函数

同时清除 cookie：

```typescript
function clearAuthStorage(): void {
  localStorage.removeItem(TOKEN_KEY)
  localStorage.removeItem(TOKEN_EXPIRY_KEY)
  removeCookieToken()
}
```

### 5. `src/features/auth/authSlice.ts` — 修改初始状态读取

确保 `initialState.token` 也能从 cookie 兜底读取（导出一个 helper 或直接调用 `getToken`）。

当前代码：
```typescript
token: localStorage.getItem('smilex_dict_token'),
```

改为从 `api.ts` 导入 `getToken` 并使用（但 `getToken` 当前未导出，需要导出或采用其他方式）。

**更好的方案**：将 cookie 读取逻辑封装到 `api.ts` 的 `getToken` 中，`authSlice.ts` 改为延迟初始化——在 `App.tsx` 的 `useEffect` 中调用 `fetchCurrentUser`，由 `getToken()` 兜底即可。`initialState.token` 保持从 `localStorage` 读取（快速初始化），若为 null 但 cookie 存在，则 `getToken()` 的兜底逻辑会在第一次 API 调用时恢复。

## 兼容性说明

- **Web**：`document.cookie` 完全兼容
- **Electron**：`document.cookie` 在 Electron renderer 中可用（即使是 sandbox 模式）
- **Capacitor (iOS/Android)**：Capacitor WebView 支持 `document.cookie`，且 cookie 比 localStorage 更持久
- **PWA**：Service Worker 环境不使用 cookie，但 PWA 运行在浏览器中，cookie 正常可用
