# 字典结构重构计划

## 一、现状分析

### 当前后端模型结构
```
dicts (字典表)
├── id, name, wordCount, source, userId

words (单词表 - 耦合了字典+用户+学习进度)
├── id, term, ipa, meaning, enMeaning, example, synonyms, synonymsNote
├── status, dictId(FK→dicts), userId(FK→users)
└── SM2字段: efactor, interval, nextReviewDate, lastReviewDate, repetitions...
```

### 核心问题
1. **单词与字典强耦合** — 每个单词通过 `dictId` 只属于一个字典，无法跨字典共享
2. **单词与用户强耦合** — 相同单词在不同用户下重复存储，无公共词库
3. **数据非规范化** — meaning/example/grammar 等全部塞在一个表的文本字段中，无法按义项/例句粒度管理
4. **官方字典硬编码** — 前端 `dictsSlice.ts` 中 CET4/CET6/TOEFL 等官方字典为静态数据

### temp/models.py 的设计参考（MySQL 版本）
- `DtDict` — 词基础信息（word_text, phonetic_uk/us, tags, level）
- `DtDictDesc` — 词性描述（properties, desc）
- `DtDictExample` — 例句（sentence, mean, ref, audio_url）
- `DtDictGrammer` — 语法（grammer_label, to_word_id, word_text）
- `DtDictParaphrase` — 释义（mean, description, description_en, scene, synonym）
- `DtDictPhrase` — 短语搭配（word_text, word_desc）

## 二、新模型设计

### 2.1 公共词库层（无用户归属，全局共享）

```python
# ---- 公共词库：单词基础 ----
class WordModel(Base):
    __tablename__ = "words"
    id           = Column(String, primary_key=True)
    term         = Column(String, nullable=False, unique=True, index=True)
    ipa          = Column(String)
    phonetic_uk  = Column(String)
    phonetic_us  = Column(String)
    phonetic_uk_url = Column(String)
    phonetic_us_url = Column(String)
    level        = Column(String)        # 难度等级：cet4/cet6/toefl 等
    tags         = Column(String)        # 逗号分隔标签
    created_at   = Column(DateTime)
    updated_at   = Column(DateTime)

# ---- 公共词库：注释/释义 ----
class WordMeaningModel(Base):
    __tablename__ = "word_meanings"
    id              = Column(String, primary_key=True)
    word_id         = Column(String, ForeignKey("words.id"), nullable=False, index=True)
    properties      = Column(String)      # 词性：n./v./adj. 等
    description     = Column(Text)        # 中文释义
    description_en  = Column(Text)        # 英文释义
    scene           = Column(String)      # 使用场景
    from_type       = Column(String)      # 来源：youdao/oxford 等
    synonym_ids     = Column(String)      # 同义词ID列表
    synonym_words   = Column(String)      # 同义词文本
    sort_order      = Column(Integer, default=0)

# ---- 公共词库：例句 ----
class WordExampleModel(Base):
    __tablename__ = "word_examples"
    id          = Column(String, primary_key=True)
    word_id     = Column(String, ForeignKey("words.id"), nullable=False, index=True)
    sentence    = Column(Text)            # 英文例句
    translation = Column(Text)            # 中文翻译
    from_type   = Column(String)          # 来源
    ref         = Column(String)          # 出处
    audio_url   = Column(String)          # 音频URL
    sort_order  = Column(Integer, default=0)

# ---- 公共词库：语法 ----
class WordGrammarModel(Base):
    __tablename__ = "word_grammars"
    id             = Column(String, primary_key=True)
    word_id        = Column(String, ForeignKey("words.id"), nullable=False, index=True)
    grammar_label  = Column(String)       # 语法标签：过去式/复数/比较级等
    word_text      = Column(String)       # 变形后的词
    to_word_id     = Column(String)       # 关联的词ID（如 原型→变形）
    sort_order     = Column(Integer, default=0)

# ---- 公共词库：短语搭配 ----
class WordPhraseModel(Base):
    __tablename__ = "word_phrases"
    id          = Column(String, primary_key=True)
    word_id     = Column(String, ForeignKey("words.id"), nullable=False, index=True)
    word_text   = Column(String)          # 短语文本
    word_desc   = Column(String)          # 短语释义
    to_word_id  = Column(String)          # 关联词ID
    sort_order  = Column(Integer, default=0)
```

## 三、新模型与旧模型对比

| 维度 | 旧模型 | 新模型 |
|------|--------|--------|
| 单词存储 | 每用户每字典重复存储 | 公共词库，全局唯一（term 唯一索引） |
| 释义结构 | 单个 text 字段 | 独立 word_meanings 表，支持多义项 |
| 例句结构 | 换行分隔的 text | 独立 word_examples 表 |
| 语法/变形 | 无 | 独立 word_grammars 表 |
| 短语搭配 | 无 | 独立 word_phrases 表 |
| 字典-单词关系 | word.dictId (N:1) | dict_words 关联表 (M:N) |
| 官方字典 | 前端硬编码 | 数据库存储，source="official" |
| 学习进度 | 混在 words 表 | 独立 user_word_progress 表 |
| 查词入库 | 直接创建 word 记录 | 先写入公共词库，再创建字典关联 + 用户进度 |

## 四、数据迁移策略

### 4.1 迁移脚本设计

```
迁移步骤：
1. 创建新表（words, word_meanings, word_examples, word_grammars, word_phrases,
   dicts_new, dict_words, user_word_progress）
2. 从旧 words 表提取去重后的 term → 写入新 words 表（公共词库）
3. 将旧 words 的 meaning/enMeaning 拆分 → 写入 word_meanings
4. 将旧 words 的 example 按换行拆分 → 写入 word_examples
5. 将旧 words 的 synonyms 拆分 → 更新 word_meanings 的 synonym_words
6. 从旧 dicts 迁移 → 新 dicts 表（source 保持 "custom"）
7. 建立旧 dict-word 关系 → dict_words 关联表
8. 从旧 words 提取学习进度 → user_word_progress
9. 重命名旧表（备份）→ 重命名新表
```

### 4.2 旧 words → 新公共词库映射

```
旧 WordItemModel.term                    → WordModel.term
旧 WordItemModel.ipa                     → WordModel.ipa
旧 WordItemModel.meaning (按换行拆分)     → WordMeaningModel (多条记录)
```

#### Helper: `_build_word_items_for_user(db, words, user_id)`
批量组装向后兼容的 `WordItem[]`：
1. 提取 `word_ids = [w.id for w in words]`
2. 批量查 `WordMeaningModel` → `dict[word_id, list[meaning]]`
3. 批量查 `WordExampleModel` → `dict[word_id, list[example]]`
4. 批量查 `UserWordProgressModel` (via `get_user_progress_for_words`) → `dict[word_id, progress]`
5. 批量查 `DictWordModel` → `dict[word_id, dict_id]` (取第一条关联)
6. 遍历 words，调用 `word_item_from_models(w, meanings[w.id], examples[w.id], progress[w.id], dict_id)` 组装

#### 端点实现

**`GET /search`** — 搜索用户的单词：
```python
pattern = f"%{q}%"
# 先找用户的 word_ids
progress_rows = db.query(UserWordProgressModel.word_id).filter(
    UserWordProgressModel.user_id == current_user.id
).subquery()
words = db.query(WordModel).filter(
    WordModel.id.in_(progress_rows),
    or_(WordModel.term.ilike(pattern), WordModel.meaning.ilike(pattern))
).limit(100).all()
# BUT: WordModel 没有 meaning 列，需要 JOIN meanings 表
# 实际实现：先搜 WordModel.term，或搜 WordMeaningModel.description 然后反向取 word_ids
```
实际搜索策略：
1. `WordModel.term.ilike(pattern)` 获取 word_ids_1
2. `WordMeaningModel.description.ilike(pattern)` 获取 word_ids_2
3. 合并去重，取交集 `& user progress word_ids`
4. 查 WordModel，通过 helper 组装返回

## 八、风险与注意事项

1. **SQLite 不支持 ALTER TABLE DROP COLUMN**（旧版本），迁移采用"创建新表 → 迁移数据 → 删旧表 → 重命名"策略
2. **前端向后兼容**：API 返回格式尽量保持扁平结构，前端改动最小化
3. **数据去重**：迁移时按 term 去重，多个用户相同的 term 只保留一条公共记录
4. **官方字典初始化**：编写种子脚本，从用户提供的词表文件批量导入到公共词库
5. **事务安全**：迁移脚本必须在事务中执行，失败可回滚
6. **种子脚本路径**：`server/scripts/seed_official_dicts.py`，读取词表文件 → 写入 words 公共词库 → 创建官方 dict → 建立 dict_words 关联
