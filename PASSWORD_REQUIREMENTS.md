# 密码强度要求

## 密码规则

### 基本要求
- 最小长度：8 个字符
- 最大长度：64 个字符

### 复杂度要求（必需）
1. **大写字母**：至少包含一个大写字母（A-Z）
2. **小写字母**：至少包含一个小写字母（a-z）
3. **数字**：至少包含一个数字（0-9）
4. **特殊字符**：至少包含一个特殊字符

**特殊字符列表**：
```
! @ # $ % ^ & * ( ) _ + - = [ ] { } | ; : , . < > ? /
```

## 示例

### 有效密码
- `Password123!` ✓
- `MySecret@2024` ✓
- `Test#Pass1` ✓

### 无效密码
- `password1` ✗（缺少大写字母和特殊字符）
- `PASSWORD1!` ✗（缺少小写字母）
- `Password!` ✗（缺少数字）
- `Password123` ✗（缺少特殊字符）
- `Pass1!` ✗（长度不足8个字符）

## 安全建议

1. **不要使用常见密码**：
   - 123456
   - password
   - qwerty
   - admin

2. **不要使用个人信息**：
   - 生日
   - 用户名
   - 手机号

3. **定期更换密码**：
   - 建议3-6个月更换一次
   - 不同网站使用不同密码

4. **使用密码管理器**：
   - 推荐使用1Password、Bitwarden等工具
   - 生成并存储强密码

## 实现细节

### 前端验证（React）
- 实时验证密码格式
- 显示友好的错误提示
- 在提交前验证

### 后端验证（FastAPI）
- Pydantic模型验证长度
- 自定义验证函数检查复杂度
- 返回清晰的中文错误信息

### 错误提示

| 错误类型 | 提示信息 |
|---------|---------|
| 长度不足 | 密码需要至少8个字符 |
| 无大写字母 | 密码需要包含大写字母 |
| 无小写字母 | 密码需要包含小写字母 |
| 无数字 | 密码需要包含数字 |
| 无特殊字符 | 密码需要包含特殊字符 |
