# Plan: Fix Import Issues & Optimize Logging

## Problem Analysis

### Issue 1: Import Dictionary "参数错误" (Parameter Error)

After thorough analysis, the root causes of the import parameter errors are:

**1a. JSON Import (`POST /api/import`) — Missing required fields cause 422 validation errors**

The `ExportData` schema requires all fields (`dicts`, `words`, `articles`, `stats`) as `List[...]` (non-optional). If an exported JSON has empty arrays or is missing fields, Pydantic validation fails with a 422 error. The current error handler does NOT handle `RequestValidationError` separately, so the user gets a generic error instead of a helpful message.

**1b. CSV Import (`POST /api/words/bulk`) — `id` field collision**

The frontend generates IDs like `import-{Date.now()}-{i}`, which could collide if imported multiple times. More critically, `WordItem` schema requires `meaning: str` (non-optional), but some CSV rows may have empty meanings, causing 422 validation failures.

**1c. TXT Import (`POST /api/import/txt` & `/api/import/quick-txt`) — Likely working but lacks error feedback**

The `FormData` sends `dictId` as a form field, which FastAPI correctly receives via `Form(...)`. However, if the dict doesn't exist or is invalid, the error is caught but poorly communicated due to the error handler issue.

**1d. The HTTPException handler hides original error messages**

The `error_handler.py` translates error details using `ERROR_TRANSLATIONS` and `STATUS_MESSAGES`, which overwrites the actual Chinese error messages from services (e.g., "仅支持 .txt 文件" gets replaced with "请求参数错误" for 400 status).

### Issue 2: Logging — No Detailed Global Exception Logging

Current problems:
- The `RequestValidationError` (422) is NOT handled — FastAPI's default handler returns raw validation errors without logging them
- The generic `Exception` handler logs via `logger.exception()` but doesn't include request context (path, method, params)
- No request trace ID for debugging
- The `RequestLoggingMiddleware` logs method/path/status/duration but doesn't log request body or query params for failed requests
- Service-level code uses `getLogger(__name__)` which produces logger names like `app.services.import_service`, but the logging config only configures the `app` logger — child loggers propagate up, but the naming is inconsistent

## Detailed Changes

### 1. `server/app/middleware/error_handler.py` — Complete Rewrite

```python
import logging
import traceback
import uuid

from fastapi import HTTPException, Request
from fastapi.exceptions import RequestValidationError
from fastapi.responses import JSONResponse

logger = logging.getLogger("app.error")

ERROR_TRANSLATIONS = {
    "Username already registered": "用户名已被注册",
    "Incorrect username or password": "用户名或密码错误",
    "Could not validate credentials": "认证失败，请重新登录",
}

STATUS_MESSAGES = {
    400: "请求参数错误",
    401: "用户名或密码错误",
    403: "没有访问权限",
    404: "资源不存在",
    422: "数据格式错误",
    500: "服务器内部错误",
}


def _is_chinese(text: str) -> bool:
    """Check if text contains Chinese characters."""
    return any('\u4e00' <= c <= '\u9fff' for c in text)


def _get_request_context(request: Request) -> dict:
    return {
        "method": request.method,
        "path": request.url.path,
        "query": str(request.query_params) if request.query_params else "",
    }


def register_exception_handlers(app):
    @app.exception_handler(RequestValidationError)
    async def validation_exception_handler(request: Request, exc: RequestValidationError):
        errors = []
        for error in exc.errors():
            field = ".".join(str(loc) for loc in error.get("loc", []))
            msg = error.get("msg", "验证错误")
            errors.append(f"{field}: {msg}")
        
        ctx = _get_request_context(request)
        logger.warning(
            f"请求参数验证失败: method={ctx['method']}, path={ctx['path']}, "
            f"query={ctx['query']}, errors={errors}"
        )
        
        first_error = errors[0] if errors else "请求参数验证失败"
        return JSONResponse(
            status_code=422,
            content={"detail": first_error},
        )

    @app.exception_handler(HTTPException)
    async def http_exception_handler(request: Request, exc: HTTPException):
        detail = exc.detail
        if isinstance(detail, str):
            if _is_chinese(detail):
                pass  # Keep original Chinese message
            else:
                detail = ERROR_TRANSLATIONS.get(
                    detail, STATUS_MESSAGES.get(exc.status_code, detail)
                )
        
        ctx = _get_request_context(request)
        if exc.status_code >= 500:
            logger.error(
                f"HTTP {exc.status_code}: method={ctx['method']}, path={ctx['path']}, "
                f"detail={detail}"
            )
        elif exc.status_code >= 400:
            logger.warning(
                f"HTTP {exc.status_code}: method={ctx['method']}, path={ctx['path']}, "
                f"detail={detail}"
            )
        
        return JSONResponse(
            status_code=exc.status_code,
            content={"detail": detail},
        )

    @app.exception_handler(Exception)
    async def general_exception_handler(request: Request, exc: Exception):
        ctx = _get_request_context(request)
        logger.error(
            f"未捕获的异常: method={ctx['method']}, path={ctx['path']}, "
            f"query={ctx['query']}, exception_type={type(exc).__name__}, "
            f"message={str(exc)}\n{traceback.format_exc()}"
        )
        return JSONResponse(
            status_code=500,
            content={"detail": "服务器错误，请稍后重试"},
        )
```

### 4. `server/app/middleware/logging.py` — Enhanced Request Logging

```python
import logging
import time
import uuid

from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
from starlette.responses import Response

logger = logging.getLogger("app.access")


class RequestLoggingMiddleware(BaseHTTPMiddleware):
    async def dispatch(self, request: Request, call_next) -> Response:
        request_id = request.headers.get("X-Request-ID", uuid.uuid4().hex[:12])
        start = time.perf_counter()
        
        response = await call_next(request)
        
        elapsed_ms = (time.perf_counter() - start) * 1000
        
        if response.status_code >= 400:
            logger.warning(
                f"{request.method} {request.url.path} "
                f"{response.status_code} {elapsed_ms:.1f}ms "
                f"[{request_id}] query={request.query_params}"
            )
        else:
            logger.info(
                f"{request.method} {request.url.path} "
                f"{response.status_code} {elapsed_ms:.1f}ms "
                f"[{request_id}]"
            )
        
        response.headers["X-Request-ID"] = request_id
        return response
```
