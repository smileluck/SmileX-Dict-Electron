# SmileX-Dict 后端重构计划

## 概述

将现有 FastAPI 后端（`server/` 目录，5 个平铺 Python 文件，~2000 行）重构为生产级模块化架构。

**用户决策**：
- API 路径保持 `/api/xxx` 不变（通过内部路由器实现版本化但不改前端）
- 数据库：保持 SQLite（优化连接池/WAL 模式）
- 部署：前后端一体化 Docker

## 二、核心模块设计

### 2.1 配置管理 (`app/config.py`)

使用 `pydantic-settings` 统一管理所有配置项，支持 `.env` 文件和环境变量：

```python
from pydantic_settings import BaseSettings

class Settings(BaseSettings):
    APP_NAME: str = "SmileX Dict"
    DEBUG: bool = False
    SECRET_KEY: str  # 必填，无默认值
    ALGORITHM: str = "HS256"
    ACCESS_TOKEN_EXPIRE_DAYS: int = 7
    DATABASE_URL: str = "sqlite:///./smilex.db"
    ALLOWED_ORIGINS: list[str] = ["*"]
    LOG_LEVEL: str = "INFO"
    RATE_LIMIT_PER_MINUTE: int = 60

    model_config = {"env_file": ".env", "env_file_encoding": "utf-8"}

settings = Settings()
```

### 2.2 应用工厂 (`app/main.py`)

```python
from fastapi import FastAPI
from app.config import settings
from app.middleware.logging import RequestLoggingMiddleware
from app.middleware.error_handler import register_exception_handlers
from app.routers import auth, dicts, words, articles, stats, settings, data, health

def create_app() -> FastAPI:
    app = FastAPI(title=settings.APP_NAME, docs_url="/docs" if settings.DEBUG else None)
    _register_middleware(app)
    _register_routers(app)
    register_exception_handlers(app)
    return app

def _register_middleware(app):
    app.add_middleware(RequestLoggingMiddleware)
    # CORS...

def _register_routers(app):
    app.include_router(auth.router,   prefix="/api/auth",     tags=["Auth"])
    app.include_router(dicts.router,  prefix="/api/dicts",    tags=["Dicts"])
    app.include_router(words.router,  prefix="/api/words",    tags=["Words"])
    app.include_router(articles.router, prefix="/api/articles", tags=["Articles"])
    app.include_router(stats.router,  prefix="/api/stats",    tags=["Stats"])
    app.include_router(settings.router, prefix="/api/settings", tags=["Settings"])
    app.include_router(data.router,   prefix="/api",          tags=["Data"])
    app.include_router(health.router, prefix="/api",          tags=["Health"])
```

### 2.3 数据库层 (`app/database.py`)

- SQLite WAL 模式提升并发
- 连接池配置
- 依赖注入 `get_db`

### 2.4 统一错误处理 (`middleware/error_handler.py`)

- 注册 HTTP 和通用异常处理器

从 router 中提取所有业务逻辑到 service 层，router 只负责：
- 参数校验（Pydantic）
- 调用 service
- 返回响应

### 2.6 日志 (`middleware/logging.py`)

- 请求/响应日志（方法、路径、状态码、耗时）
- 结构化 JSON 日志（生产环境）
- 使用标准 `logging` 模块

### 2.7 限流 (`middleware/rate_limit.py`)

- 基于 IP 的简易内存限流
- 防止暴力破解登录接口

## 三、性能优化

1. **SQLite WAL 模式**：`PRAGMA journal_mode=WAL` 提升读写并发
2. **批量导入优化**：`bulk_create_words` 改为批量 `db.add_all()` + 单次 `db.commit()`
3. **数据库索引**：为 `words.term`、`words.userId`、`words.dictId` 添加复合索引
4. **连接池**：SQLite 使用 `StaticPool` 或 `pool_size` 配置
5. **异步查询**：搜索接口添加 `LIMIT` 防止返回过多结果

---

## 四、安全加固

1. **SECRET_KEY 强制配置**：无默认值，缺失时启动报错
2. **密码验证**：提取为 Pydantic validator（当前在 router 里重复验证）
3. **输入清理**：搜索关键词过滤 SQL 特殊字符（SQLAlchemy 参数化已防护，额外防御）
4. **JWT 增强**：Token 添加 `iat`（签发时间）字段
5. **CORS 收紧**：生产环境不允许 `*`
6. **文件上传限制**：限制上传文件大小（如 5MB）
7. **速率限制**：登录/注册接口限流
8. **响应头**：添加安全头（X-Content-Type-Options, X-Frame-Options 等）

```dockerfile
COPY . .
RUN pnpm build

# Stage 2: 后端 + Nginx
```

### docker-compose.yml

```yaml
services:
  app:
    build: .
    ports:
      - "80:80"
    env_file: .env
    volumes:
      - ./data:/app/data  # SQLite 数据持久化
```

### CI/CD (`.github/workflows/ci.yml`)

```yaml

on: [push, pull_request]
jobs:
  backend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/checkout@v4

  docker:

    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: docker build -t smilex-dict .
```
