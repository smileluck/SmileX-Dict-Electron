# 自定义词典编辑功能 - 实施计划

## 概述

在现有 SmileX Dict 项目中新增独立的词典编辑页面 `/dicts/:id/edit`，提供完整的词典管理能力：词典信息编辑、词典内单词的增删改查、手动添加单词（含在线查词补全信息）。

## 设计方案

### 新增路由

- `GET /dicts/:id/edit` — 词典编辑页面（需认证），仅允许编辑 `source=custom` 的自建词典

### 页面布局

```
┌─────────────────────────────────────────────┐
│ ← 返回词典中心   |   词典名称（可编辑）      │
│ 词典描述（可编辑）                            │
├─────────────────────────────────────────────┤
│ [+ 手动添加] [在线查词添加] [导入]  🔍 搜索   │
├─────────────────────────────────────────────┤
│  # | 单词   | 音标     | 释义     | 操作     │
│  1 | apple  | /ˈæp.əl/ | n. 苹果  | ✏️ 🗑️   │
│  2 | banana | /bəˈnæn.ə/| n. 香蕉 | ✏️ 🗑️   │
│  ...                                        │
├─────────────────────────────────────────────┤
│        分页 或 加载更多                       │
└─────────────────────────────────────────────┘
```

### 交互流程

1. **进入页面**: 从 Dicts.tsx 的自建词典卡片点击"编辑"按钮跳转
2. **编辑词典信息**: 点击词典名称/描述进入编辑模式，回车或失焦保存
3. **手动添加单词**: 弹窗输入 term + meaning + ipa + example 等字段，提交到 `POST /api/words`
4. **在线查词添加**: 弹窗输入英文单词，调用 `GET /api/words/lookup`，展示结果后确认添加到词典
5. **编辑单词**: 点击✏️图标弹出编辑弹窗，可修改 meaning/ipa/example/synonyms 等，调用 `PUT /api/words/:id`
6. **删除单词**: 点击🗑️图标弹出确认框，从词典移除调用 `DELETE /api/words/:id`
7. **搜索过滤**: 输入关键词实时过滤当前词典中的单词列表

## 实施步骤

### Step 1: 后端 — 添加词典单词列表分页查询 API

**文件**: `server/app/routers/words.py`

现有 `GET /api/words?dictId=xxx` 已能按词典查询单词列表，但缺少分页支持。增加 `page` 和 `pageSize` 查询参数：

```python
@router.get("", response_model=List[WordItem])
def list_words(
    dictId: Optional[str] = None,
    page: int = Query(default=1, ge=1),
    pageSize: int = Query(default=50, ge=1, le=200),
    current_user: UserModel = Depends(get_current_user),
    db: Session = Depends(get_db),
):
    ...
    # 返回分页后结果
```

同时添加一个新的响应头 `X-Total-Count` 或在返回值中包含总数，以便前端显示分页信息。

### Step 2: 前端 — 在 api.ts 中补充 API 方法

**文件**: `src/services/api.ts`

- 扩展 `dictsApi.update` 支持 `description` 字段
- 扩展 `wordsApi.list` 支持 `page` / `pageSize` 参数
- 已有的 `wordsApi.create` / `wordsApi.update` / `wordsApi.delete` 无需修改

### Step 3: 前端 — 新增词典编辑页面组件

**文件**: `src/routes/DictEdit.tsx`（新建）

组件结构：
- 顶部：返回按钮 + 词典名称（行内编辑）+ 词典描述（行内编辑）+ 保存/取消
- 工具栏：手动添加按钮、在线查词添加按钮、导入按钮、搜索框
- 单词表格：序号、term、ipa、meaning、操作列（编辑/删除）
- 底部：分页 或 "加载更多" 按钮
- 弹窗组件：
  - `AddWordModal`: 手动添加单词表单（term, meaning, ipa, example, synonyms）
  - `EditWordModal`: 编辑单词表单（同上，预填数据）
  - `LookupAddModal`: 在线查词后添加（复用现有 lookup API）

### Step 4: 前端 — 注册路由

**文件**: `src/App.tsx`

在路由表中添加：
```tsx
<Route path="/dicts/:id/edit" element={isAuthenticated ? <DictEdit /> : <Navigate to="/login" ... />} />
```

### Step 5: 前端 — 修改 Dicts.tsx 添加编辑入口

**文件**: `src/routes/Dicts.tsx`

在自建词典卡片的操作按钮区域，添加"编辑"按钮（非 special 词典才显示）：
```tsx
<button onClick={() => navigate(`/dicts/${d.id}/edit`)}>编辑</button>
```

### Step 6: 国际化

**文件**: `src/i18n/locales/zh.json` 和 `src/i18n/locales/en.json`

添加 `dictEdit` 命名空间下的翻译 key：
- `dictEdit.title` / `dictEdit.back` / `dictEdit.dictName` / `dictEdit.dictDesc`
- `dictEdit.addWord` / `dictEdit.lookupAdd` / `dictEdit.search`
- `dictEdit.editWord` / `dictEdit.deleteWord` / `dictEdit.confirmDelete`
- `dictEdit.term` / `dictEdit.meaning` / `dictEdit.ipa` / `dictEdit.example` / `dictEdit.synonyms`
- `dictEdit.saveSuccess` / `dictEdit.deleteSuccess` / `dictEdit.noWords`
