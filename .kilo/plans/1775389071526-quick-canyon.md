# Plan: Add i18n (中文/English) Support for Frontend

## Overview

Add internationalization (i18n) to the SmileX Dict React frontend using `react-i18next` + `i18next`, supporting **Chinese (zh)** and **English (en)**.

## Tech Choice

- **i18next** + **react-i18next**: Industry-standard i18n solution for React projects
- **i18next-browser-languagedetector**: Auto-detect browser language
- Language files: JSON format, organized by locale

## Steps

### 1. Install dependencies

```bash
npm install i18next react-i18next i18next-browser-languagedetector
```

### 2. Create i18n configuration (`src/i18n/index.ts`)

- Initialize i18next with:
  - `localStorage` lookup for saved preference (key: `smilex-dict-lang`)
  - Browser language detection as fallback
  - Default language: `zh`
  - Fallback language: `zh`
  - Load translation resources inline (no async loading needed for 2 small locales)

### 3. Create translation files

#### `src/i18n/locales/zh.json` (Chinese - extract all existing Chinese text)
#### `src/i18n/locales/en.json` (English translations)

Organize translations into namespaces/sections:
- `nav` - Navigation labels (主页, 词典, 书籍, 更多, 登录, etc.)
- `home` - Home page text
- `login` - Login/Register page text
- `settings` - Settings page text
- `common` - Shared text (加载中..., 保存中..., etc.)
- `practice` - Practice-related pages
- `dicts` - Dictionary pages
- `library` - Library/Book pages
- `panel` - Dashboard/Panel pages
- `about` - About page
- `collections` - Collections/WrongWords/Mastered pages
- `vocabAnalysis` - Vocabulary analysis pages

### 4. Initialize i18n in `src/main.tsx`

- Import `./i18n` at the top of main.tsx (side-effect import initializes i18next)

### 5. Add language switch to Settings page (`src/routes/Settings.tsx`)

- Add a new "语言 / Language" section in Settings
- Provide zh/en toggle buttons
- Use `i18next.changeLanguage()` to switch language
- Language preference is auto-persisted via i18next-browser-languagedetector (localStorage)

```

  i18n/
    index.ts              # i18next configuration
    locales/
      zh.json             # Chinese translations
      en.json             # English translations
  main.tsx                # + import './i18n'
  App.tsx                 # + useTranslation

  components/

```

## Notes

- No changes to Redux store needed — language preference is managed by i18next's built-in localStorage persistence
- The approach is incremental: Chinese translations will exactly match current hardcoded text; English translations will be added alongside
